# A.O.G.D Records

Публичная база ориентировок A.O.G.D (Agency Of Good Deeds) с отдельной административной панелью.

## Что уже работает

- поиск и фильтры по статусу;
- карточки и подробные досье;
- загрузка и автоматическое уменьшение фотографий;
- создание, редактирование и удаление записей;
- закрытый вход по секретному паролю;
- постоянное хранение записей в Cloudflare D1 и фотографий в Cloudflare R2;
- локальный демо-режим без сервера;
- адаптация для телефона и компьютера.

Админ-панель открывается по адресу `/#/admin`.

## Запуск на компьютере

Нужен Node.js 20.19 или новее.

```bash
pnpm install
pnpm dev
```

При обычном запуске через Vite серверная часть Cloudflare не работает, поэтому сайт автоматически включает локальный демо-режим. При первом входе в админ-панель придумайте пароль длиной от 8 символов. Данные останутся только в текущем браузере.

## Публикация через GitHub и Cloudflare Pages

GitHub Pages подходит только для демонстрации: он не запускает серверные Functions, поэтому общая база и настоящий закрытый пароль там работать не будут. Для рабочего сайта используйте бесплатный Cloudflare Pages, подключённый к GitHub.

### 1. Загрузите проект в GitHub

Создайте пустой репозиторий и загрузите в него всё содержимое этой папки. Файл `.dev.vars` и секретный пароль в репозиторий не добавляйте.

### 2. Создайте проект Cloudflare Pages

В панели Cloudflare откройте **Workers & Pages**, создайте Pages-проект и подключите GitHub-репозиторий.

Параметры сборки:

- Framework preset: `React (Vite)`;
- Build command: `pnpm build` (или `npm run build`);
- Build output directory: `dist`;
- Root directory: `/`.

После первого развёртывания Cloudflare будет автоматически публиковать каждое изменение основной ветки GitHub. Официальная инструкция: [Git integration](https://developers.cloudflare.com/pages/configuration/git-integration/) и [Build configuration](https://developers.cloudflare.com/pages/configuration/build-configuration/).

### 3. Создайте базу D1

В Cloudflare создайте D1 database с любым именем, например `aogd-records`. Откройте её SQL Console и выполните содержимое файла `migrations/0001_init.sql`.

Затем в настройках Pages-проекта добавьте D1 binding:

- Variable name: `DB`;
- D1 database: созданная база `aogd-records`.

Альтернативно миграцию можно применить через Wrangler командой `npx wrangler d1 migrations apply aogd-records --remote`. Официальное описание миграций: [D1 migrations](https://developers.cloudflare.com/d1/reference/migrations/).

### 4. Создайте хранилище фотографий R2

Создайте R2 bucket, например `aogd-media`. В настройках Pages-проекта добавьте R2 binding:

- Variable name: `MEDIA`;
- R2 bucket: созданное хранилище `aogd-media`.

R2 bucket не нужно делать публичным: фотографии отдаёт защищённый маршрут самого сайта. Официальная инструкция по привязкам: [Pages Functions bindings](https://developers.cloudflare.com/pages/functions/bindings/).

### 5. Установите пароль администратора

В настройках Pages-проекта откройте **Variables and Secrets**, добавьте значение:

- Name: `ADMIN_TOKEN`;
- Value: длинный уникальный пароль (лучше 24–40 случайных символов);
- Type: **Secret / Encrypt**.

Не используйте пароль от Telegram, GitHub или Cloudflare. Создайте отдельный пароль только для этой панели. Официальная инструкция: [Cloudflare Pages secrets](https://developers.cloudflare.com/pages/functions/bindings/#secrets).

### 6. Повторно разверните проект

После добавления `DB`, `MEDIA` и `ADMIN_TOKEN` запустите новый deployment. На главной исчезнет предупреждение о демо-режиме, а в панели появится сообщение «Постоянное хранилище подключено».

## Безопасность и публикация данных

- Публично видны все записи из базы. Административные операции требуют `ADMIN_TOKEN` на сервере.
- Пароль хранится только как зашифрованный секрет Cloudflare и временно в `sessionStorage` вкладки администратора.
- Фотографии автоматически конвертируются в WebP и ограничены размером 5 МБ после обработки.
- Не публикуйте персональные данные, обвинения или фотографии без проверенного законного основания и согласованной процедуры удаления/обжалования.
- A.O.G.D — самостоятельный проект. Не представляйте его как официальный ресурс INTERPOL, Telegram или государственного органа.

## Где менять внешний вид

Основной интерфейс находится в `src/App.jsx`, стили — в `src/styles.css`, эмблема — в `public/aogd-emblem.png`. Компоненты React Bits можно будет добавить поверх этой структуры, не меняя API, базу и админ-панель.
